프로젝트 개요

김치 프리미엄(Korea Premium) 차익거래 자동화 봇의 수익을 토큰화하여, 사전 합의된 소수 투자자에게 투명하고 검증 가능한 방식으로 투자 참여 기회를 제공하는 프라이빗 투자 플랫폼이다.

핵심 컨셉

DeFi 프로토콜 UI/UX를 차용하되, 실제로는 프라이빗 투자 구조. 투자자에게 익숙한 APY · 토큰 · 온체인 검증 언어를 사용해 신뢰감과 투명성을 확보한다.

투자자 신뢰 확보온체인 토큰 + 수동 오라클눈에 보이는 자산 = 심리적 안도감
투명성 제공업비트 읽기전용 API 공개조작 불가 구조로 의심 원천 차단

운영 단순화수동 승인 기반 워크플로우소수 투자자 규모에 적합

서비스 구조

전체 자금 흐름

1
투자자 → 운영팀 계좌 현금 입금
투자계약서 서명 완료 후 지정 계좌로 KRW 입금. 거래소 AML 이슈 없음.
2
운영팀 → 업비트 입금 → 봇 운용
기존 합법적 자금(3.5억)과 동일 루트. 입금 출처 소명 일관성 유지.
3
봇 수익 발생 → 토큰 바이백/소각
봇 수익의 일정 비율(예: 50%)로 시장에서 토큰 바이백 후 소각. 나머지는 재투자 + 준비금 적립.
4
토큰 희소성 증가 → 가격 상승
유통량 감소 → 토큰 가격 상승. 오라클로 가격 수동 업데이트. 투자자 대시보드에 반영.
5
투자자 출금 요청 → 운영팀 바이백 → 현금 송금
투자자가 토큰을 운영팀 지갑으로 전송 → 관리자 승인 → 현재 토큰 가격 기준 KRW 송금.

프레이밍 전략

핵심

"토큰 판매"가 아닌 "투자계약 + 토큰 무상 증정" 프레이밍. 현금은 투자계약서 기반으로 수취하고, 토큰은 그 계약의 증표로 무상 제공. 법적으로 "토큰 판매업" 프레이밍 회피.

오라클 구조

설계 원칙

실시간 자동화 오라클 대신, 수동 업데이트 + 투명성 보완 구조. 개발 난이도 낮추면서 신뢰도 확보.

레이어방법목적
1차 (일상적)관리자가 수동으로 토큰 가격 업데이트편의성
2차 (선택적 검증)업비트 읽기전용 API 키 투자자에게 공유완전 검증 — 서버 미경유
3차 (기록)월간 수익 스냅샷 해시값 온체인 기록위변조 불가 증명

업비트 API 키 설정

권한 설정:
  ✅ 자산 조회 (읽기)
  ❌ 주문 생성/취소
  ❌ 출금
  
→ 투자자에게 공유해도 안전. 잔고만 조회 가능.
⚠ 수동 오라클 신뢰 확보

수동이지만 투자자가 언제든 API 키로 직접 검증 가능 → "조작 가능하다"는 의심 자체가 사라지는 구조. 오라클 업데이트 후 투자자 전체에게 공지 루틴 권장.

Prisma Schema

// prisma/schema.prisma
generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}

model User {
  id            String   @id @default(cuid())
  email         String?  @unique
  walletAddress String?  @unique
  name          String?
  createdAt     DateTime @default(now())
  investment    Investment?
  withdraws     WithdrawRequest[]
}

model Investment {
  id           String   @id @default(cuid())
  userId       String   @unique
  user         User     @relation(fields: [userId], references: [id])
  amountKrw    Int      // 투자금 (원)
  tokenAmount  Decimal  // 발행 토큰 수
  entryPrice   Decimal  // 취득 단가
  status       InvestmentStatus @default(PENDING)
  contractSigned Boolean @default(false)
  depositConfirmed Boolean @default(false)
  tokenMinted  Boolean @default(false)
  createdAt    DateTime @default(now())
  updatedAt    DateTime @updatedAt
}

model OraclePrice {
  id        String   @id @default(cuid())
  price     Decimal  // KRW 기준 토큰 가격
  updatedBy String   // 관리자 ID
  note      String?  // 업데이트 사유
  createdAt DateTime @default(now())
}

model WithdrawRequest {
  id          String   @id @default(cuid())
  userId      String
  user        User     @relation(fields: [userId], references: [id])
  tokenAmount Decimal
  krwAmount   Int      // 지급 예정 금액
  txHash      String?  // 토큰 전송 트랜잭션
  status      WithdrawStatus @default(PENDING)
  createdAt   DateTime @default(now())
  processedAt DateTime?
}

model InviteLink {
  id        String   @id @default(cuid())
  token     String   @unique @default(cuid())
  usedBy    String?
  usedAt    DateTime?
  expiresAt DateTime?
  createdAt DateTime @default(now())
}

enum InvestmentStatus { PENDING DEPOSITED ACTIVE WITHDRAWN }
enum WithdrawStatus   { PENDING APPROVED COMPLETED REJECTED }
